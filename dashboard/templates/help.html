Post Page

All Social Media posts which are being tracked would be shown here.

User can also filter or search posts based upon different filters i.e. username, likes, post date etc.

Searching Metrics:

  1. Usernames:
    A user can easily filter out the posts based upon the username. The powerful search engine also allows the user to search posts from multiple users at the same time using comma separated values.
  2. Number of likes:
    Posts can also be categorized based upon the number of likes. One can easily provide the range of likes to filter out the posts.
  3. Number of comments:
    Number of comments can also be used to filter out the posts.
  4. Number of shares:
    Another metric is the number of shares which can be used to easily filter out the posts.
  5. Post date:
    Last metric is the post date which provides convinience to the user to filter out the posts based upon the post date.

Also in addition, one can use all of these filters at the same time to do more robust search.

Each post displays the metrics which could be used to analyze the overall engagement on any social media post. That includes the no. of likes, comments, shares and overall sentiment of the comments on that post.

For each post, a user can have the following interactions:

  1. The user can view the post details by clicking on the post
  2. The user can capture the data manually for the post.It allows the user to add data manually just in case scrapper stops working or would like to enter post metadata manually on an ad-hoc basis
  3. It allows to manually edit the data for the post. For now one can edit the frequency (the time the scrapper will take to extract updated metrics).
  4. The user can delete any post from the system at any time after the confirmation.

New Post to Track

Here, the user can add a new post to track in the system.

The user is required to provide the POST ID, Social Media platform, and frequency of the scrapper to add new post in the system.

The system provides an additional feature to test scrapper before the user add any post to track. The user can test the scrapper to see if it is working fine or not.

A unique ID is generated with each post which could help to search posts from the DB.

Add Entry Manually

A user can also manually add data of any specific post in the system. For this, the user is required to provide the following information:

Example:

  1. Number of likes
  2. Number of shares
  3. Comments etc

Edit Entry Manually

In the Posts page, all the available posts that are been tracked are shown here.

The search engine is powerful in a way that one is allowed to search for posts by different criteria that are connected to the post.

Example:

  1. Usernames
  2. # of likes
  3. # of comments
  4. date
  5. # of likes
  6. # of comments

Post Detail View Page

The page should you:

  1. the date of the Post
  2. URL to the Post
  3. the content itself
  4. graphs with time series
  5. the unique ID generated based on static metrics, and that would serve as an index for fast search

Metrics shown in the graph are:

  1. number of likes
  2. number of shares
  3. number of comments
  4. sentiment analysis of comments
  5. media type e.g. if the Post is an image/video
  6. Post’s longevity e.g. time it took for the post to “die”
